What is a water year?

The year in the dataset and the water year are different in that the year column represents calendar year, while the wy column represents the water year. A water year is defined as starting on October 1st of one year and ending on September 30th of the next year. The 2025 water year in the dataset starts on October 1st, 2024 and ends on September 30th, 2025.

A water year is a period of time defined by the US Geological Survey to account for precipitation falling in fall/winter that accumulates as snow and thus doesn’t leave a watershed until the following spring/summer. At NCOS, it makes sense to consider trends in the time period of water years because the amount of rain that falls in the fall/winter is a strong determinant of water availability etc. for the spring and summer. The cycle starts again with rain in the fall and winter, which would mark the start of the next water year.

3. Cleaning the data

This dataset includes a lot of columns and rows that we don’t need.

We can use functions from packages in the tidyverse to help us clean the data.

First, we could use filter() from the dplyr package in the tidyverse to filter the dataset to only include dates from the 2025 water year.

water1 <- filter(
  venoco_water, # starting with venoco_water
  wy == "2025" # only include observations from 2025 water year
)

Then, we could use mutate() (also from dplyr) to create a new column in which the values are water elevations converted to meters.

water2 <- mutate(
  water1, # starting with water1 data frame
  comp_level_m = (comp_level_ft + 2.96) * 0.3048 # adding logger height and converting to meters from ft
)

Then, we could use select() (also from dplyr) to choose the columns we need to answer our guiding question.

water3 <- select(
  water2, # starting with water2 data frame
  date, comp_level_m, temperature # select columns of interest
)

4. The pipe operator

Creating multiple data frames (water1, water2, water3) as we did above can be tedious. Instead, you can use the pipe operator |> to chain functions together.

You can think of the pipe operator as being a way of saying “and then”.

In the code we will write, we could say that we will:

  • create a new object called venoco_clean from venoco_water and then…
  • filter to only include wy that matches 2025 and then…
  • mutate to create a new column called comp_level_m to add logger elevation to comp_level_ft and convert to meters and then…
  • select the columns of interest (only date, comp_level_m, and temperature)

This is “pseudocode”. Pseudocode looks different depending on the coding language, but in this class, “pseudocode” will refer to a bulleted list of steps you will later write in code.

Writing pseudocode makes working with data much more straightforward. If you can plan out your code in plain language, it is easier to write the code (and troubleshoot when things go wrong!).

# create a new object from venoco_water
venoco_clean <- venoco_water |> 
  # filter to only include 2025 water year
  filter(wy == "2025") |> 
  # create a new column to convert water elevation in ft to meters
  mutate(comp_level_m = (comp_level_ft + 2.96) * 0.3048) |> 
  # select columns of interest
  select(date, comp_level_m, temperature)

5. Grouping data

In this dataset, each row is an observation of water level elevation at a particular time on a given day.

What if you want to visualize the mean water level for a given day?

You can add a function called group_by() from the dplyr package which allows you to “group” a data frame.
When you use group_by(), you are essentially telling R: there are “groups” in this data frame that I need to pay attention to.

When you pair group_by() with summarize() (also from dplyr), you can calculate summary statistics (e.g. calculate the mean daily water elevation) by each group (the date).

In this next section, we will:

  • create a new object called venoco_daily from venoco_clean
  • group by date to
  • summarize (calculate) mean daily water level in m
  • ungroup the dataframe
# create new object from venoco_clean
venoco_daily <- venoco_clean |> 
  # group data frame by date
  group_by(date) |> 
  # calculate mean water level in meters for a given day
  summarize(mean_level_m = mean(comp_level_m, na.rm = TRUE)) |> 
  # ungroup the data frame
  ungroup()

6. Visualizing data

To answer our question (How does mean daily water surface elevation change over the course of a single water year at Venoco Bridge?), we need to visualize the data in a timeseries plot.

Timeseries plots have time on the x-axis and the variable of interest on the y-axis.

To visualize the data, we’ll use ggplot() and its associated functions, which are all in the ggplot2 package in the tidyverse.

The main parts of creating a plot in ggplot() are:

  1. the global call: telling R that you want to use ggplot() by naming the function
  2. the data and mapping: telling R the data frame you want to use (with the argument data =) and the columns in that data frame to plot (with the argument mapping = and the aes() function for the “aesthetics”)
  3. the geometries: these are the shapes, lines, etc. in the plot. In this case, we’ll use two geometries: geom_point() to represent mean daily water elevation and geom_line() to represent connections between each day.
# base layer: ggplot
ggplot(data = venoco_daily,
       # x-axis should be date
       # y-axis should be mean daily water elevation
       mapping = aes(x = date,
                     y = mean_level_m)) +
  # first layer: points
  geom_point(color = "#287948") +
  # second layer: line
  geom_line(color = "#287948") +
  # relabeling x- and y-axes, adding title
  labs(x = "Date",
       y = "Mean daily water elevation (m)",
       title = "Water year 2025 (01 October 2024 - 30 September 2025)") +
  # using theme that is different from default
  theme_bw()

## 6. Visualizing data
