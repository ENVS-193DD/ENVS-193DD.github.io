Summary

Guiding questions

What are the differences in mean salinity between sites in the 2024 water year?

How do sites differ in salinity through time in the 2024 water year?

How does dissolved oxygen stratification differ through time at Venoco Bridge for the 2024 and 2025 water years?

Data science tools

To answer these questions, we are going to:

  • review reading in packages
    • library()
  • use familiar packages
    • tidyverse
  • introduce new packages
    • janitor to clean column names
    • here to manage file paths
  • review reading in data
    • read_csv()
  • review cleaning and wrangling functions/operators
    • filter()
    • mutate()
    • select()
    • case_when()
    • fct_relevel()
    • %in%
  • review summarizing functions
    • group_by(), summarize(), and ungroup()
    • count()
  • review visualizations using ggplot(), including:
    • geom_point() and geom_line()
    • geom_bar()
    • facet_wrap()
    • scale_color_manual()
  • introduce a new visualization tool:
    • stat_summary() from ggplot2 (within tidyverse) to visualize quick summary statistics
  • introduce new tools to deal with dates and times from the lubridate package (within tiydverse):
    • mdy_hms() and mdy() to parse dates into POSIXct (Portable Operating System Interface calendar time) format which is a standard way of denoting dates and times
    • month() to extract the month from a given POSIXct object
    • year() to extract the year from a given POSIXct object
  • introduce a new wrangling concept: joins using functions from the dplyr package (within tidyverse)
    • left_join() to join two data frames (x and y) and retain all rows in x
    • full_join() to join two data frames (x and y) and retain all rows in x and y
  • introduce new wrangling functions:
    • pull() from the dplyr package to extract a data frame column as a vector
    • relocate() from the dplyr package to rearrange columns in a data frame
    • rename() from the dplyr package to rename a column
  • introduce a new operator:
    • ! to exclude observations
  • introduce new ways of formatting Quarto documents, including
    • callout blocks
    • in-line code

2. cleaning

cleaning weather

Goal

Data frame with the following columns:

  • date
  • precipitation, max temperature, min temperature
  • month, year
  • water year

Pseudocode

  • create new object from weather
  • clean column names
  • make sure that date is understood as date
  • select columns of interest
  • extract month and year from date
  • make new column for water year

Code

# creating new object from weather
weather_clean <- weather |> 
  # cleaning column names
  clean_names() |> 
  # making sure date is in POSIXct format
  mutate(date = mdy(date)) |> 
  # selecting columns of interest
  select(date, prcp, tmax, tmin) |> 
  # extracting month and year from date column
  mutate(month = month(date),
         year = year(date)) |> 
  # assigning water year
  mutate(wy = case_when(
    # when month is October or later, assign value of year + 1
    month >= 10 ~ year + 1,
    # when month is September or earlier, assign value of existing year
    .default = year))

cleaning metadata

Goal

Data frame with the following columns:

  • ArcGIS Survey123 info:

    • global ID,
    • creation date,
    • edit date

  • observation date and time:

    • monitoring date (note the format of this column),
    • time of wq measurement

  • observation context:

    • weather,
    • flow,
    • notes,
    • water surface elevation

  • observation location:

    • site name,
    • other site name,
    • x,
    • y

Pseudocode

  • create new object from metadata
  • clean column names
  • exclude unneeded columns
  • make sure monitoring date is understood as date
  • rename monitoring date column to show that it has datetimes
  • extract month and year from monitoring date
  • create new column for water year

Code

# creating new object from metadata
metadata_clean <- metadata |> 
  # cleaning column  names
  clean_names() |> 
  # excluding unnecessary columns
  select(!c(object_id, creator, editor)) |> 
  # making sure monitoring_date is in POSIXct format
  mutate(monitoring_date = mdy_hms(monitoring_date)) |> 
  # renaming monitoring datetime column to reflect values
  rename(monitoring_datetime = monitoring_date) |> 
  # creating new columns for month and year of observations
  mutate(month = month(monitoring_datetime),
         year = year(monitoring_datetime)) |> 
  # assigning water year
  mutate(wy = case_when(
    # when month is October or later, assign value of year + 1
    month >= 10 ~ year + 1,
    # when month is September or earlier, assign value of existing year
    .default = year)) |> 
  # create new column of full site name
  mutate(site_full = case_when(
    site_name == "east_channel" ~ "East Channel",
    site_name == "phelps_bridge" ~ "Phelps Bridge",
    site_name == "venoco_bridge" ~ "Venoco Bridge",
    site_name == "other" ~ "Other"
  ))

Consider the survey protocol

You can get information about the survey protocol from the Survey123 form.

In the “Depth code” field, the description is: Enter the correct code value for the depth you are monitoring, depending on the total depth of the site. For example, if the site si 2 feet deep, enter 2 for the surface, 1 for 1 foot depth, and 0 for the bottom. Bottom measurements will always be 0. If you are only able to take a surface measurement, enter 1.

Draw this out on a piece of paper. If a survey has depth codes of 0, 1, 2, 3, and 4, where would those fall relative to the bottom?

  • parameters:

    • DO (mg/L)
    • DO (%sat)
    • Conductivity (uS/cm)
    • Salinity (ppt)
    • Temperature (°C)

Pseudocode

  • create new object from water parameters
  • clean column names
  • exclude unneeded columns
  • rename depth code to be elevation

Code

# creating new object from water_parameters
water_parameters_clean <- water_parameters |> 
  # cleaning object names
  clean_names() |> 
  # excluding unnecessary columns
  select(!c(creator, editor)) |> 
  # renaming columns
  rename(elevation_code = depth_code) |> 
  # replacing incorrect elevation code from 2021-07-16
  mutate(elevation_code = case_when(
    elevation_code == 10 ~ 1,
    .default = elevation_code
  )) |> 
  # set factors
  mutate(elevation_code = as_factor(elevation_code),
         elevation_code = fct_relevel(elevation_code,
                                      "0", "1", "2", "3", "4", "5", "6"))

4. Joining data frames

left join: keeps all observations in x

right join: keeps all observations in y

full join: keeps all observations in x and y

water quality = metadata + water parameters

# join metadata and water parameters
water_quality <- full_join(
  # left
  x = metadata_clean, 
  # right
  y = water_parameters_clean,
  # shared column
  by = c("global_id" = "parent_global_id")) |> 
  # stop and look at colnames(water_quality) before continuing
  # selecting relevant columns
  select(
    # information about observation
    global_id, monitoring_datetime, time_of_wq_measurement, 
    monitor_name, weather, site_name, other_site_name, site_full, x, y, 
    
    # water information
    water_surface_elevation, flow, notes, 
    
    # measurement information
    elevation_code, do_mg_l, do_percent_sat, conductivity_u_s_cm,
    salinity_ppt, temperature_c) |> 
  # extract date from monitoring datetime
  mutate(date = date(monitoring_datetime)) |> 
  # move date column to appear to the right of monitoring_datetime
  relocate(date, .after = monitoring_datetime) |> 
  # exclude monitoring_datetime
  select(!monitoring_datetime)

water quality weather = water quality + weather

# join water quality and weather
water_quality_weather <- full_join(
  # left
  x = water_quality, 
  # right

# left join (exclude rows that don't match in weather)
water_quality_weather <- left_join(

5. Visualizations

Site differences in mean salinity

Guiding question

What are the differences in salinity between sites in the 2024 water year?

Planning

Goal: create a plot to compare differences in salinity between sites:

  • x-axis should be site
  • y-axis should be salinity
  • points should represent individual salinity measurements at each site

Necessary wrangling steps:

  • filter outliers
  • exclude “other” sites
  • filter to only include observations from 2024 water year
  • filter to only include observations from bottom, ~1 ft above bottom, ~2 ft above bottom
  • reorder site levels

Code

Wrangling:

# create new object from water_quality_weather
salinity_filtered <- water_quality_weather |> 
  # filter out outliers
  filter(!(global_id %in% salinity_outlier_ids)) |> 
  # filter out other sites
  filter(site_full != "Other") |> 
  # only include observations from 2024 water year
  filter(wy == "2024") |> 
  # filter bottom, ~ 1ft above bottom, ~2ft above bottom
  filter(elevation_code %in% c("0", "1", "2")) |> 
  # reorder site levels from lowest to highest mean salinity
  mutate(site_full = fct_relevel(
    site_full,
    "Phelps Bridge", "East Channel", "Venoco Bridge"
  ))

Quick summary:

# start with filtered data frame
salinity_filtered |>

Necessary cleaning steps:

Code

Wrangling:

# start with joined data frame
venoco_do <- water_quality_weather |> 
  # filter to only include Venoco,
  filter(site_full == "Venoco Bridge" &
           # water years of interest
           wy %in% c("2024", "2025") &
           # bottom, ~ 1 ft, ~ 2 ft above bottom
           elevation_code %in% c("0", "1", "2")) |> 
  # select columns of interest
  select(date, wy, do_mg_l, elevation_code) |> 
  # relevel elevation code order
  mutate(elevation_code = fct_relevel(elevation_code,
                                      "2", "1", "0"))

Stratification is more obvious in the 2025 water year than in the 2024 water year.

The magnitude of stratification is greater in winter than in spring and summer, though there is some stratification in summer even though DO is lower overall.

### cleaning weather

#### Goal

#### Pseudocode

- create new object from weather

- make sure that date is understood as date
- select columns of interest

#### Code

```{r}
#| label: cleaning-weather-data

# creating new object from weather
weather_clean <- weather |> 
  # cleaning column names
  clean_names() |> 
  # making sure date is in POSIXct format
  mutate(date = mdy(date)) |> 
  # selecting columns of interest
  select(date, prcp, tmax, tmin) |> 
  # extracting month and year from date column
  mutate(month = month(date),
         year = year(date)) |> 
  # assigning water year
  mutate(wy = case_when(
    # when month is October or later, assign value of year + 1
    month >= 10 ~ year + 1,
    # when month is September or earlier, assign value of existing year
    .default = year))
```
